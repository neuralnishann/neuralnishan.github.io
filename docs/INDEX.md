## 🎓 Learning Paths

### Path 1: I want to add content (Blog/Project/Page)
1. Read [QUICK_REFERENCE.md](QUICK_REFERENCE.md)
2. Read "Writing Content" in [DEVELOPMENT.md](DEVELOPMENT.md)
3. Create your content
4. Reference examples in [source/](../source/) folder

**Time:** ~10 minutes

### Path 2: I want to change the design
1. Read [CUSTOMIZATION_GUIDE.md](../CUSTOMIZATION_GUIDE.md)
2. Explore [ARCHITECTURE.md](ARCHITECTURE.md#-theme-architecture)
3. Edit colors in `assets/css/modern-professional.css`
4. Reference [MODERNIZATION_GUIDE.md](../MODERNIZATION_GUIDE.md)

**Time:** ~20 minutes

### Path 3: I'm new and want to understand everything
1. Read [README.md](../README.md) - Overview
2. Read [PROJECT_STRUCTURE.md](PROJECT_STRUCTURE.md) - Where things are
3. Read [ARCHITECTURE.md](ARCHITECTURE.md) - How it works
4. Read [DEVELOPMENT.md](DEVELOPMENT.md) - How to develop
5. Read [CONTRIBUTING.md](CONTRIBUTING.md) - How to contribute

**Time:** ~60 minutes

### Path 4: I want to deploy or troubleshoot
1. Read [QUICK_REFERENCE.md](QUICK_REFERENCE.md#-troubleshooting)
2. Read "Troubleshooting" in [DEVELOPMENT.md](DEVELOPMENT.md#-troubleshooting)
3. Check specific issue sections below

**Time:** ~5 minutes

### Path 5: I want to understand the build system
1. Read [PROJECT_STRUCTURE.md](PROJECT_STRUCTURE.md) - Directory layout
2. Read "Build Process" in [ARCHITECTURE.md](ARCHITECTURE.md#-build-process-details)

**Time:** ~15 minutes
